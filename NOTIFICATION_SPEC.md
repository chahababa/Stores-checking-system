# 通知機制規格 v1

## 1. 目的

第一版通知先做站內通知中心，不先接 Email 或 LINE。

目標是讓使用者在系統內就能立即看到：

- 高風險異常
- 本月加強與連續低分
- 待改善任務是否堆積

## 2. 優先級

- 高：必查項目出現 C、客訴項目出現 C、整店總評為 C
- 中：本月加強項目仍落在 B / C、連續低分兩週以上、待改善任務過多
- 低：本月巡店摘要或本月尚未巡店

## 3. 顯示位置

- `/notifications` 提供完整通知中心
- 首頁提供通知摘要預覽

## 4. 後續擴充

- 第二版可加入已讀 / 未讀
- 第三版再接 Email / LINE
